Introduction

In Machine Learning we must teach the machine, or the machine must otherwise learn. We use a lot of math to do this, graphics cards, and some friendly help from google. Google Colabs is a colabrative labratory for public use. Which means it's free, but you might not always get the best. Colabs allows users to remotely access a computer system and run code on it. These computer systems have access to power graphic processor untis or tensor processing units, both of which we can use, but we will be using GPUs.


Coin Sorting

I'm going to describe Machine Learning as machine see, machine do. To start with you give your machine a bunch of data, pictures, sensor inputs, imdb ratings, what have you, and ask it to tell you what category it falls into. After it tells you its guess you tell it how correct or incorrect it was. It then tries again, changing it's equations slightly to do a better job. The math sort of looks like this Y=W1X1+W2X2+B where X1 and X2 are the input variables (the brightness of a pixel, the length of a fish, the distance to an object) and W1 and W2 are weights and B is a bias. The machine will slowly change W1, W2, and B till it has a line where everything on one side is only one thing, or close enough. More inputs? More Xs. Think of the process as a coin sorter bank, every coin has to go into the proper spot, so it builds a coin sorter, tests how well it did, and then rearranges the parts to test again. This repeats until it's accurate enough. This is our model.


Accuracy and Dataset

Our model is graded on it's accuracy and it will generally improve over time. It is possible for it to get too good, we call this overfitting. It has trained for so long on our data that it can't properly sort new data. To work around this we split our data into three categories, train, validate, and test. The training data is used to train the model, the validatation data is used to test the train results, and the test is used to test our end model so we can say look how good we've done.
